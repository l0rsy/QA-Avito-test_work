## Задание 1: Скриншот с багами

![](./tests_images/task_1.png)

### Предусловие

**Скриншот не дает однозначного понимания, в каком состоянии находится поиск**

**Вариант 1:** Поиск был сделан <u>без</u> применения фильтров\
В таком случае непонятно, почему объявлений 8, а не 19 **(см. баг №5)**

**Вариант 2:** Поиск был сделан <u>с</u> применением фильтров\
В таком случае непонятно, почему кнопка активна и почему есть объявления с ценой, не входящей в нужный диапазон **(см. баги №2, №3, №4)**

В связи с неоднозначным условиями скришота баги помечены как **"ситуативные"**

### Баги

### P0 (критический)

1) Поиск ведется по городу **Москва**, а среди объявлений есть объявление из **Новосибирска**

2) **(СИТУАТИВНЫЙ БАГ, см. Предусловие)** Отображается 2 объявления, цена которых выходит за указанные диапазоны. У одного меньше 1000 руб (500 руб), у другого более 30 000 руб (40 000 руб)\
В этом баге мы предполагаем, что фильтры все-таки были применены, а значит цены точно должны входить в диапазон

3) **(СИТУАТИВНЫЙ БАГ, см. Предусловие)** Отображается объявление **<u>"Персидский котёнок девочка"</u>**, хотя фильтр **"Пол" = "Мальчик"**\
В этом баге мы предполагаем, что фильтры все-таки были применены, а фильтр пола должен учитываться\
Также нужно уточнить по ТЗ работы сайта, должны ли отображаться надписи "мальчик" у объявлений, где проставлена эта характеристика. Без этого возникает неопределенность, подходят ли нам остальные объявления вообще

4) **(СИТУАТИВНЫЙ БАГ, см. Предусловие)** Отображается объявление, с котом с породой **"Экзотический"**\
В этом баге мы предполагаем, что фильтры все-таки были применены, а должны отображаться только коты с породой "Персидский"

### P1 (высокий)

5) **(СИТУАТИВНЫЙ БАГ, см. Предусловие)** Результат поиска отображается как **"Котята в Москве 19"**, а объявлений всего 8 (не 19)\
Однозначно определить баг невозможно, не хватает данных. Возможно объявления ниже тоже относятся к результату поиска, но в таком случае не должно быть надписи "Подобрали для вас"

6) Справа над объявлениями текст **"Ошибка, обновите страницу"** выглядит как техническое сообщение, но размещен в интерфейсе, как статический объект

7) В левом верхнем углу надпись **"Коръера в Авито"** содержит орфографическую ошибку

8) Тумблер **"Сначала из Можайска"** некорректен, так как пользователь не выбрал этот район (область) в блоке справа от поиска\
Нужно уточнить по ТЗ работы сайта, нужна ли такая кнопка в целом

### P2 (средний)

9) При наведении на кнопку в виде сердца справа от названия объявления появляется надпись **"Не показывайте мне похожие объявления"**\
Логичнее было бы отображать надпись **"Показывать мне похожие объявления"**

10) Слева наверху (ниже логотипа Авито) навигационная цепочка **"Главная>Животные><u>Персидская</u>>Кошки"** содержит логическую ошибку.\
Должно быть **"Главная>Животные><u>Кошки</u>>Персидская"**

11) Только у **2/8** объявлений отображается время с того момента, как они были выложены\
Если такая функция существует, то явно должна работать на каждом объявлении

12) Предложенный фильтр **"Щенок"** не имеет смысла, так как поиск ведется по кошкам\
Этого фильтра не должно быть уже не шаге, когда пользователь выбрал в навигационной цепочке "Кошки"

### P3 (низкий)

13) У объявления **"Экзотический короткошёрстный котёнок"** рядом с ценой есть какой-то значок\
Интуитивно непонятно, что он означает. Скорее всего скидку. Если это так, то лучше отображать цену, которая была и которая стала теперь\
В любом случае нужно уточнить по ТЗ, что означает стрелка вниз в черном кружке

## Задание 2.2: Тесты UI 







## Инструкция по запуску тестов для задания 2.2

- Node.js 18+ (лучше LTS)
- npm 9+

### Установка
1) Установить зависимости:

```bash
npm install
```
Удостовериться, что устанвоились браузеры
```bash
npx playwright install
```

### Использование вечного пользователя в тестах

Тесты логинятся в существующий аккаунт через ручку и прокидывают авторизацию в localStorage.

Для использования СВОЕГО пользователя нужно ввести его данные в файле .env

```bash
USER_EMAIL=email
USER_PASSWORD=password
```

### Запуск тестов

Запустить все тесты:

```bash
npx playwright test
```

Запустить конкретный тестовый файл reateAccount.test.ts:
```bash
npx playwright test tests/accountTests/createAccount.test.ts
```

Запустить конкретный тест по названию:
```bash
npx playwright test -g "Открытие страницы Мои объявления"
```
